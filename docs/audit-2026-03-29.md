## Detailed Findings by Severity

### P0 — Blocking

**[P0] Viewport blocks user zoom**
- **Location:** `index.html:8`
- **Category:** Accessibility
- **Impact:** Users who need to zoom (low vision, motor impairment) cannot. Fails WCAG 2.1 SC 1.4.4.
- **WCAG:** 1.4.4 Resize Text (AA)
- **Recommendation:** Change `user-scalable=no` to `user-scalable=yes`

**[P0] Hardcoded grid columns break mobile**
- **Location:** `src/pages/Seasons.tsx:37` (`cols={3}`), `src/pages/Players.tsx:11` (`cols={4}`)
- **Category:** Responsive Design
- **Impact:** Cards are unreadably small on phone screens. Content overflows or is crushed.
- **Recommendation:** Use responsive objects: `cols={{ base: 1, sm: 2, md: 3 }}`

### P1 — Major

**[P1] Navigation links are semantically wrong**
- **Location:** `src/components/Navbar/Navbar.tsx:64-103`
- **Category:** Accessibility
- **Impact:** Login link (`<a href="#">` with onClick) and user info display (`<a>` with `preventDefault`) confuse screen readers. Logout link navigates nowhere.
- **WCAG:** 4.1.2 Name, Role, Value (A)
- **Recommendation:** Use `<button>` for Login/Logout actions. Use `<div>` or `<Text>` for display-only user info.

**[P1] GitHub button has wrong href**
- **Location:** `src/components/Home/Home.tsx:43-56`
- **Category:** Anti-Pattern
- **Impact:** `href` points to `https://github.com/mantinedev/mantine` (template leftover) while `onClick` opens the correct repo. Right-click "open in new tab" goes to wrong URL.
- **Recommendation:** Fix `href` to match `onClick` target, or remove `onClick` and just use `href`.

**[P1] Footer avatar missing alt text**
- **Location:** `src/components/Footer/Footer.tsx:30`
- **Category:** Accessibility
- **Impact:** Screen readers announce the image with no context.
- **WCAG:** 1.1.1 Non-text Content (A)
- **Recommendation:** Add `alt="Jeff Probst"` or `alt=""` if decorative.

**[P1] Admin operations have no user feedback**
- **Location:** `src/pages/Admin.tsx` (all upload functions)
- **Category:** Accessibility / UX
- **Impact:** Clicking upload buttons shows nothing -- success logs to console, errors are swallowed. Users don't know if it worked.
- **Recommendation:** Add Mantine `notifications` for success/error states.

**[P1] Hardcoded season filter in view layer**
- **Location:** `src/pages/Seasons.tsx:40`
- **Category:** Anti-Pattern
- **Impact:** Every new season requires a code change to this filter string. Easy to forget.
- **Recommendation:** Add a `visible` or `active` boolean to the season data model.

### P2 — Minor

**[P2] Console.logs left in production code**
- **Location:** `src/hooks/useSeason.ts:13`, `src/hooks/useCompetition.ts:44-45`, `src/hooks/useEvents.ts:39`, `src/pages/Seasons.tsx:39`
- **Category:** Performance
- **Impact:** Pollutes DevTools, minor perf cost on every render/snapshot.
- **Recommendation:** Remove or gate behind `import.meta.env.DEV`.

**[P2] No route-level code splitting**
- **Location:** `src/AppRoutes.tsx:21-28`
- **Category:** Performance
- **Impact:** All pages (Admin, Draft, SeasonAdmin) load upfront even if user never visits them.
- **Recommendation:** Use `React.lazy()` + `Suspense` for page imports.

**[P2] Custom font "Greycliff CF" never loaded**
- **Location:** `src/components/Home/Home.module.css:22-23`
- **Category:** Theming
- **Impact:** Falls back to Mantine default font. The `font-family` declaration is dead code.
- **Recommendation:** Either load the font or remove the reference.

**[P2] Empty theme.ts**
- **Location:** `src/theme.ts`
- **Category:** Theming
- **Impact:** No custom brand colors, typography, or component defaults. Relies entirely on Mantine defaults.
- **Recommendation:** Define primary color, font, and default component props.

**[P2] `Button role="link"` on Get Started**
- **Location:** `src/components/Home/Home.tsx:32`
- **Category:** Accessibility
- **Impact:** A `<Button>` with `role="link"` is semantically confusing. If it navigates, use a link. If it acts, use a button.
- **Recommendation:** Remove `role="link"` or use `<Anchor>` / `<Link>` component.

## Patterns & Systemic Issues

- **Semantic HTML gap:** Interactive elements consistently use `<a href="#">` instead of `<button>` -- this is a pattern across the Navbar, not a one-off.
- **No user feedback pattern:** All Firestore write operations (admin uploads, episode creation, game events) log to console but show nothing to the user.
- **Hardcoded visibility:** Season visibility and admin access rely on string comparisons and client-side checks scattered across components.

## Positive Findings

- **CSS Modules with Mantine tokens:** All CSS files use `light-dark()`, `var(--mantine-*)` variables, and Mantine breakpoint vars. Dark mode works out of the box.
- **Responsive typography:** Hero title and description scale down properly at `sm` breakpoint.
- **AppShell layout:** Good use of Mantine's AppShell with collapsible mobile navbar and burger menu.
- **Table scroll containers:** Admin CRUD tables use `TableScrollContainer minWidth={300}` preventing horizontal overflow.
- **Footer responsive flex:** Correctly switches to column layout on xs breakpoint.
